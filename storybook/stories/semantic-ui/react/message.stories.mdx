# Message

A message displays information that explains nearby content.

## Standard

### Default

A basic message.

<Preview>
    <Story name="standard">
        <Message>
            <Message.Header>
    Setup a security policy for external sharing
            </Message.Header>
            <p>
    Easily manage external sharing links and set-up automatic access reviews
    for entrusted group owners.
            </p>
        </Message>
    </Story>
</Preview>

### List

A message with a list.

<Preview>
    <Story name="list">
        <Message>
            <Message.Header>New Site Features</Message.Header>
            <Message.List>
                <Message.Item>You can now have cover images on blog pages</Message.Item>
                <Message.Item>Drafts will now auto-save while writing</Message.Item>
            </Message.List>
        </Message>
    </Story>
</Preview>

### Dismissible

Dismissible blocks do not automatically close. Use the onDismiss callback to remove the message.

<Preview>
    <Story name="dismissible">
        <Message onDismiss={noop}>
            <Message.Header>New Site Features</Message.Header>
            <Message.List>
                <Message.Item>You can now have cover images on blog pages</Message.Item>
                <Message.Item>Drafts will now auto-save while writing</Message.Item>
            </Message.List>
        </Message>
    </Story>
</Preview>

## Two Columns Layout

A message can have two columns

### Left Aligned

A message can have a button or an icon positionned to the left.

<Preview>
    <Story name="two columns left aligned">
        <Message icon>
            <Button>Click Here</Button>
            <Message.Content>
                <Message.Header>Just one second</Message.Header>
                <p>We are fetching that content for you.</p>
            </Message.Content>
        </Message>
    </Story>
</Preview>

### Right Aligned

A message can have a button or an icon positionned to the right.

<Preview>
